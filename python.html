MODULE 2.2

Conditions (if/elif/else)

Conditions let your code make decisions. Python uses if, elif (else-if), and else. Indentation (4 spaces) defines the block - no curly braces needed.

Comparison operators
== != > < >= <= - compare two values, return bool
Logical operators
and, or, not - combine conditions
Membership
in, not in - check if value is in a collection
Ternary (inline)
value_if_true if condition else value_if_false
# Basic if/elif/else salary = 75000 if salary > 80000: role = "Senior Engineer" elif salary > 60000: role = "Engineer" else: role = "Junior Engineer" # Logical operators if salary > 60000 and is_active: print("Eligible for bonus") # Membership test approved_depts = ["Engineering", "Data", "Analytics"] if department in approved_depts: print("Access granted") # Ternary expression (one-liner) label = "Senior" if salary > 80000 else "Standard"

⚡ PySpark Relevance

Python if/elif/else translates to PySpark's when() and otherwise() - the equivalent of SQL CASE WHEN:

from pyspark.sql import functions as F df = df.withColumn("role", F.when(F.col("salary") > 80000, "Senior Engineer") .when(F.col("salary") > 60000, "Engineer") .otherwise("Junior Engineer") )

# Basic for loop over a list salaries = [50000, 75000, 90000, 60000] for salary in salaries: print(salary) # range() - generates a sequence of numbers for i in range(5): # 0, 1, 2, 3, 4 print(i) # enumerate() - index + value together for i, name in enumerate(["Alice", "Bob", "Carol"]): print(f"{i}: {name}") # zip() - iterate two lists in parallel names = ["Alice", "Bob"] sals = [95000, 80000] for name, sal in zip(names, sals): print(f"{name} earns {sal}") # List comprehension - concise way to build a list squared = [x**2 for x in range(5)] # [0, 1, 4, 9, 16] high_sal = [s for s in salaries if s > 70000] # [75000, 90000] # while loop - runs until condition is False count = 0 while count < 3: print(f"attempt {count}") count += 1 # break / continue for x in range(10): if x == 3: continue # skip 3 if x == 6: break # stop at 6 print(x)

⚡ PySpark Relevance

Loops in Python are used to build PySpark pipelines dynamically - you rarely loop over rows (PySpark handles that in parallel). Common pattern: loop to create multiple columns or aggregate multiple fields.

# Loop to add multiple transformed columns cols_to_upper = ["first_name", "last_name", "department"] for col_name in cols_to_upper: df = df.withColumn(col_name, F.upper(F.col(col_name))) # List comprehension to select specific columns numeric_cols = [c for c, t in df.dtypes if t in ('int', 'double')] df.select(numeric_cols).show()

Functions

Functions are reusable blocks of code. In data engineering, you'll write functions to transform data, validate schemas, build pipelines, and create PySpark UDFs.

Key concepts: def keyword, parameters vs arguments, default parameters, *args (variable positional), **kwargs (variable keyword), return, lambda.

# Basic function definition def greet(name): return f"Hello, {name}!" # Default parameter def get_bonus(salary, rate=0.10): return salary * rate print(get_bonus(95000)) # 9500.0 (uses default rate) print(get_bonus(95000, 0.15)) # 14250.0 # *args - accepts any number of positional arguments def sum_salaries(*args): return sum(args) print(sum_salaries(50000, 75000, 90000)) # 215000 # **kwargs - accepts any number of keyword arguments def create_employee(**kwargs): return kwargs emp = create_employee(name="Bob", dept="Data", salary=80000) # lambda - anonymous one-liner function double = lambda x: x * 2 classify = lambda s: "Senior" if s > 80000 else "Standard" # Multiple return values (returns a tuple) def stats(data): return min(data), max(data), sum(data) / len(data) lo, hi, avg = stats([50000, 75000, 90000])

⚡ PySpark UDFs (User Defined Functions)

Python functions become PySpark UDFs - custom transformations applied to every row of a DataFrame. Use sparingly; built-in PySpark functions are faster.

from pyspark.sql.functions import udf from pyspark.sql.types import StringType # Define Python function def classify_salary(salary): if salary > 80000: return "Senior" elif salary > 60000: return "Mid" else: return "Junior" # Register as UDF classify_udf = udf(classify_salary, StringType()) # Apply to DataFrame column df = df.withColumn("band", classify_udf(F.col("salary")))

# Define a class class Employee: # Class variable (shared by all instances) company = "TechCorp" # __init__ is the constructor - runs when object is created def __init__(self, name, salary, dept): self.name = name # instance variable self.salary = salary self.dept = dept # Instance method def get_band(self): if self.salary > 80000: return "Senior" return "Standard" def apply_raise(self, pct=0.10): self.salary *= (1 + pct) return self.salary def __repr__(self): return f"Employee({self.name}, {self.salary})" # Create instances alice = Employee("Alice", 95000, "Data") bob = Employee("Bob", 80000, "Eng") print(alice.get_band()) # Senior alice.apply_raise(0.15) # 109250.0 # Inheritance - extend a class class Manager(Employee): def __init__(self, name, salary, dept, reports): super().__init__(name, salary, dept) # call parent __init__ self.reports = reports def team_size(self): return len(self.reports) carol = Manager("Carol", 120000, "Data", [alice, bob])

PySpark is deeply object-oriented. When you write spark.read.csv(), you're calling methods on a SparkSession object. DataFrames are objects with methods like .filter(), .groupBy(), .withColumn().

INTERVIEW

Interview Questions

Common Python questions asked in data engineering interviews. Click each question to reveal the answer.

01 What is the difference between a list and a tuple in Python?
Easy ▾
List is mutable (can be changed after creation) and uses square brackets []. Tuple is immutable (cannot be changed) and uses parentheses ().

When to use which:
• Use a list when the collection may change - adding columns, filtering rows, building dynamic SELECT lists.
• Use a tuple when data is fixed - a row record, a coordinate pair, or a function returning multiple values.

Key difference in PySpark: DataFrame .dtypes returns a list of tuples like [('name', 'string'), ('age', 'int')] - the outer list is mutable, each inner tuple is an immutable record.
02 How does Python handle mutable vs immutable types?
Medium ▾
Immutable types: int, float, str, bool, tuple, frozenset. Their value cannot change after creation - Python creates a new object when you "modify" them.

Mutable types: list, dict, set, most objects. They can be modified in-place.

The gotcha: When you pass a mutable type to a function, the function can modify the original:
def add_column(cols, new_col): cols.append(new_col) # modifies the ORIGINAL list! my_cols = ["name", "age"] add_column(my_cols, "salary") print(my_cols) # ["name", "age", "salary"] ← changed!
This is called passing by reference. To avoid side effects, pass a copy: cols.copy() or list(cols).
03 What are *args and **kwargs?
Easy ▾
*args collects any number of positional arguments into a tuple.
**kwargs collects any number of keyword arguments into a dict.
def log_event(*args, **kwargs): print("Events:", args) # tuple of positional args print("Metadata:", kwargs) # dict of keyword args log_event("start", "transform", table="employees", rows=5000) # Events: ('start', 'transform') # Metadata: {'table': 'employees', 'rows': 5000}
In data engineering, **kwargs is commonly used to pass optional configuration parameters through multiple function layers without explicitly listing every parameter.
04 Explain list comprehensions vs generator expressions.
Medium ▾
List comprehension [expr for x in iterable] - evaluates eagerly, stores all values in memory immediately.

Generator expression (expr for x in iterable) - evaluates lazily, yields one value at a time. Uses much less memory for large datasets.
# List comprehension - all values in memory at once squares = [x**2 for x in range(1_000_000)] # 8MB in memory # Generator - compute one at a time squares_gen = (x**2 for x in range(1_000_000)) # ~120 bytes! for sq in squares_gen: process(sq) # only one value in memory at a time
Rule of thumb: Use a list comprehension when you need to index or reuse the results. Use a generator when you only iterate once over large data.
05 What is a lambda function and when would you use it?
Easy ▾
A lambda is an anonymous (unnamed) function defined in a single expression. Syntax: lambda args: expression.

Use it for short, throwaway functions - typically as arguments to sorted(), map(), filter(), or similar higher-order functions:
# Sort employees by salary descending employees = [{"name": "Alice", "salary": 95000}, {"name": "Bob", "salary": 80000}] sorted_emps = sorted(employees, key=lambda e: e["salary"], reverse=True) # map() to transform a list salaries = [50000, 75000, 90000] after_raise = list(map(lambda s: s * 1.1, salaries)) # filter() to filter a list high_sal = list(filter(lambda s: s > 70000, salaries))
Avoid using lambda for complex logic - use a named def function instead for readability.
06 How does Python manage memory? What is garbage collection?
Hard ▾
Python uses reference counting as its primary memory management strategy. Every object has a reference count - the number of variables pointing to it. When the count reaches zero, the memory is freed immediately.

Garbage collector (gc module): Handles reference cycles (object A points to B, B points back to A - reference counts never reach zero). Python's cyclic garbage collector runs periodically to detect and free these cycles.
import sys, gc x = [1, 2, 3] print(sys.getrefcount(x)) # 2 (x + getrefcount arg) y = x # reference count → 3 del y # reference count → 2 del x # reference count → 1 (then 0 → freed) # Force garbage collection (rarely needed) gc.collect()
In data engineering: Python memory management matters less when using PySpark - data lives on the JVM/cluster. But be careful with .collect() which pulls all data to the Python driver.
07 What is the difference between shallow copy and deep copy?
Medium ▾
Shallow copy: Creates a new object but references the same nested objects. Changes to nested items affect both copies.

Deep copy: Creates a completely independent copy, including all nested objects.
import copy original = [["Alice", 95000], ["Bob", 80000]] shallow = original.copy() # or list(original) deep = copy.deepcopy(original) # Modify nested item original[0][1] = 99999 print(shallow[0][1]) # 99999 - shallow copy IS affected! print(deep[0][1]) # 95000 - deep copy is NOT affected
In pipeline code, use deep copy when you need to branch transformations from the same base state without interference.
08 When would you use a set vs a list?
Easy ▾
Use a set when:
• You need to check membership fast - O(1) vs list's O(n)
• You want to eliminate duplicates automatically
• You need set operations: union, intersection, difference

Use a list when:
• Order matters
• You need index access (cols[0])
• Duplicates are valid
# Fast deduplication raw_ids = [101, 102, 101, 103, 102, 104] unique_ids = list(set(raw_ids)) # [101, 102, 103, 104] # Membership check - set is much faster for large data valid_depts = {"Engineering", "Data", "Analytics", "Operations"} if emp_dept in valid_depts: # O(1) hash lookup process(employee)
09 What are Python decorators? Give a data engineering example.
Hard ▾
A decorator is a function that wraps another function to add behavior before/after it runs - without modifying the original function's code. Uses the @ syntax.
import time, functools # Define a decorator def timer(func): @functools.wraps(func) def wrapper(*args, **kwargs): start = time.time() result = func(*args, **kwargs) # call the original elapsed = time.time() - start print(f"{func.__name__} took {elapsed:.2f}s") return result return wrapper # Apply decorator with @ syntax @timer def run_pipeline(df): return df.filter("salary > 50000").groupBy("dept").count() # Now run_pipeline() automatically times itself result = run_pipeline(df) # prints: run_pipeline took 3.42s
Data engineering uses: logging, retry logic, timing, schema validation, caching results. Airflow's @dag and @task are decorators.
10 How would you handle exceptions in a data pipeline?
Medium ▾
Use try/except/else/finally blocks to handle errors gracefully, log them, and decide whether to fail fast or continue.
import logging logger = logging.getLogger(__name__) def read_source(spark, path, table_name): try: df = spark.read.parquet(path) row_count = df.count() logger.info(f"Read {row_count:,} rows from {table_name}") return df except Exception as e: logger.error(f"Failed to read {table_name} from {path}: {e}") raise # re-raise so the pipeline fails loud finally: # Always runs - good for cleanup, closing connections logger.info(f"Finished read attempt for {table_name}") # Specific exception types try: result = int(user_input) except ValueError: print("Not a valid integer") except ZeroDivisionError: print("Cannot divide by zero") except Exception as e: print(f"Unexpected: {e}") raise
Best practice: Catch specific exceptions rather than bare except:. Always re-raise unexpected errors in production pipelines - silent failures are worse than loud ones.
