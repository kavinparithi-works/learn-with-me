Apache Spark + PySpark

From single machines to distributed clusters - process petabytes of data with the world's fastest big data engine.

Why Big Data? Why Spark?

Traditional databases and pandas work great for megabytes. When your data grows to gigabytes and terabytes, you need a new approach.

Spark vs Hadoop MapReduce: Hadoop MapReduce writes intermediate results to disk after every step. Spark keeps data in memory between steps - making it 10-100x faster for iterative algorithms like machine learning.

Parallelism Rule: If you have 100 partitions and 20 CPU cores, Spark runs 20 tasks in parallel, cycling through all 100 partitions. More partitions = more parallelism (up to number of cores).

RDDs - Resilient Distributed Datasets

RDD is the original Spark abstraction - an immutable, distributed collection of objects partitioned across the cluster. DataFrames are built on top of RDDs.

Data split into Partitions across Workers
1 million rows split into 4 partitions
Partition 0
rows 1-250k
Partition 1
rows 250k-500k
Partition 2
rows 500k-750k
Partition 3
rows 750k-1M
Each partition processed independently on a different machine

RDD Properties

  • Resilient: If a partition is lost, Spark can recompute it from the lineage (the DAG of transformations)
  • Distributed: Data lives across multiple nodes in the cluster
  • Dataset: A collection of records - any Python/Java/Scala objects
  • Immutable: You can't modify an RDD - you create new ones via transformations
  • Lazy: Transformations are not executed until an action is called
RDD Basics (rarely used directly - prefer DataFrames)
# Create RDD from Python list
rdd = sc.parallelize([1, 2, 3, 4, 5, 6, 7, 8], numSlices=4)

# Transformations (lazy)
doubled = rdd.map(lambda x: x * 2)
evens = rdd.filter(lambda x: x % 2 == 0)
words = rdd.flatMap(lambda x: [x, x*10])  # one-to-many

# Actions (trigger execution)
result = doubled.collect()  # [2,4,6,8,10,12,14,16]
total = rdd.reduce(lambda a, b: a + b)  # 36
count = rdd.count()  # 8
first = rdd.first()  # 1

# Check partitions
print(rdd.getNumPartitions())  # 4
In Practice: You'll rarely use RDDs directly. DataFrames and Datasets (the structured APIs) are 90%+ of real-world PySpark. They're faster because Spark's Catalyst optimizer can optimize them.

DAG and Lazy Evaluation

Spark builds a Directed Acyclic Graph (DAG) of all your transformations. Nothing runs until you call an action. This lets Spark optimize the entire plan before executing.

How Spark Builds and Executes a DAG
Stage 1 (no shuffle)
Read CSV
→
filter(year=2024)
→
withColumn(total)
Shuffle Boundary (groupBy triggers network exchange)
groupBy(country)
Stage 2 (after shuffle)
Partial Aggregation
→
agg(sum, count)
→
ACTION: show()
Only when show() is called does Spark actually run all stages. Until then, it just builds the graph.

Transformations vs Actions

Transformations (Lazy)
  • select(), filter(), where()
  • withColumn(), drop()
  • groupBy(), orderBy()
  • join(), union()
  • map(), flatMap() (RDD)
  • distinct(), repartition()
Actions (Trigger Execution)
  • show(), display()
  • collect(), take(n)
  • count(), first()
  • write.save()
  • reduce(), foreach()
  • toPandas()

PySpark Setup and SparkSession

SparkSession is the entry point to everything in PySpark. One SparkSession per application.

Creating a SparkSession
from pyspark.sql import SparkSession
from pyspark.sql import functions as F
from pyspark.sql.types import *
from pyspark.sql.window import Window

# Create SparkSession
spark = SparkSession.builder \
    .appName("MyApp") \
    .master("local[*]") \   # use all CPU cores locally
    .config("spark.executor.memory", "4g") \
    .getOrCreate()

# In Databricks, spark is already available - no need to create
print(spark.version)  # 3.5.0

# SparkContext (lower level, for RDDs)
sc = spark.sparkContext

# Stop when done (not needed in Databricks)
# spark.stop()
Reading Data
# Read CSV
df = spark.read \
    .option("header", "true") \
    .option("inferSchema", "true") \
    .csv("/data/sales.csv")

# Read with explicit schema (recommended for production)
schema = StructType([
    StructField("id", IntegerType(), nullable=False),
    StructField("name", StringType(), nullable=True),
    StructField("amount", DoubleType(), nullable=True),
    StructField("date", DateType(), nullable=True)
])
df = spark.read.schema(schema).csv("/data/sales.csv")

# Read Parquet (most common in production)
df = spark.read.parquet("/data/sales.parquet")

# Read JSON
df = spark.read.json("/data/events.json")

# Read Delta (requires Delta Lake)
df = spark.read.format("delta").load("/data/sales_delta")

# Create from in-memory list
data = [("Alice", 1000), ("Bob", 2000)]
df = spark.createDataFrame(data, ["name", "salary"])

DataFrames - The Core API

A DataFrame is a distributed table with named columns and a schema. It's the primary way to work with structured data in PySpark.

Exploring DataFrames

df = spark.createDataFrame([

Transformations

Transformations create new DataFrames from existing ones. They are lazy - recorded but not executed until an action is called.

Select and Filter
# Select columns
df.select("name", "salary").show()
df.select(F.col("name"), F.col("salary") * 1.1).show()

# Filter rows (WHERE equivalent)
df.filter(F.col("salary") > 80000).show()
df.where("dept = 'Engineering'").show()  # SQL string syntax
df.filter((F.col("dept") == "Engineering") & (F.col("salary") > 90000))

# Add/update columns
df = df.withColumn("salary_k", F.col("salary") / 1000)
df = df.withColumn("bonus", F.when(F.col("dept") == "Engineering", F.col("salary") * 0.2)
                                .when(F.col("dept") == "Marketing", F.col("salary") * 0.15)
                                .otherwise(F.col("salary") * 0.1))

# Rename and drop
df = df.withColumnRenamed("salary", "annual_salary")
df = df.drop("hire_date")

# Sort
df.orderBy(F.col("salary").desc()).show()
df.sort("dept", F.col("salary").asc_nulls_last()).show()
Aggregations
# GroupBy and aggregate
df.groupBy("dept") \
  .agg(
      F.count("*").alias("headcount"),
      F.avg("salary").alias("avg_salary"),
      F.sum("salary").alias("total_salary"),
      F.max("salary").alias("max_salary"),
      F.min("salary").alias("min_salary"),
      F.collect_list("name").alias("members")
  ).show()

# Pivot
df.groupBy("dept") \
  .pivot("year", [2022, 2023, 2024]) \
  .agg(F.sum("revenue")).show()

# Multiple groupBy keys
df.groupBy("dept", "location").agg(F.count("*")).show()
String, Date, and Null Functions
# String functions
df.withColumn("upper_name", F.upper(F.col("name")))
df.withColumn("name_len", F.length(F.col("name")))
df.withColumn("starts_a", F.col("name").like("A%"))
df.withColumn("trimmed", F.trim(F.col("name")))
df.withColumn("replaced", F.regexp_replace(F.col("email"), "@.*", ""))

# Date functions
df.withColumn("year", F.year(F.col("hire_date")))
df.withColumn("month", F.month(F.col("hire_date")))
df.withColumn("days_employed", F.datediff(F.current_date(), F.col("hire_date")))
df.withColumn("next_review", F.date_add(F.col("hire_date"), 365))

# Null handling
df.filter(F.col("salary").isNull())
df.filter(F.col("salary").isNotNull())
df.fillna({"salary": 0, "dept": "Unknown"})
df.dropna(subset=["salary", "dept"])
df.withColumn("salary", F.coalesce(F.col("salary"), F.lit(0)))

Actions - Triggering Execution

Actions trigger computation. Use them carefully - calling collect() on a huge DataFrame will crash your driver.

Common Actions
# Display (triggers job)
df.show()                     # top 20 rows
df.show(50, truncate=False)   # 50 rows, full text

# Count
df.count()  # full scan - expensive on large tables!

# Collect - DANGER on large data
rows = df.collect()           # returns List[Row] to driver
first_row = df.first()        # first row only
top5 = df.take(5)             # first 5 rows

# Convert to Pandas (small data only)
pdf = df.limit(1000).toPandas()

# Write
df.write.mode("overwrite").parquet("/output/results")
df.write.mode("append").format("delta").save("/delta/table")
df.write.mode("overwrite").saveAsTable("catalog.schema.table")

# Write CSV with options
df.coalesce(1) \
  .write.option("header", "true") \
  .mode("overwrite") \
  .csv("/output/report.csv")
Performance Tip: Never call collect() on DataFrames with millions of rows. It brings all data to the driver and will run out of memory. Use show(), count(), or write to storage instead.

Spark SQL

Spark SQL lets you run ANSI SQL queries directly on DataFrames by registering them as temporary views. Same optimizer, different syntax.

Using SQL with DataFrames
# Register DataFrame as a temp view
df.createOrReplaceTempView("employees")

# Run SQL
result = spark.sql("""
    SELECT
        dept,
        COUNT(*) AS headcount,
        AVG(salary) AS avg_salary,
        MAX(salary) AS max_salary
    FROM employees
    WHERE salary > 60000
    GROUP BY dept
    ORDER BY avg_salary DESC
""")
result.show()

# Mix SQL and DataFrame API
dept_stats = spark.sql("SELECT * FROM employees WHERE dept = 'Engineering'")
dept_stats \
    .withColumn("seniority", F.when(F.col("salary") > 90000, "Senior").otherwise("Junior")) \
    .show()

# Global temp view (visible across sessions)
df.createOrReplaceGlobalTempView("global_employees")
spark.sql("SELECT * FROM global_temp.global_employees")

Spark Joins

Joins in Spark work like SQL joins but are distributed. Understanding Broadcast Join vs Sort-Merge Join is critical for performance.

Broadcast Join vs Sort-Merge Join
Broadcast Join (small + large)
Small table copied to ALL executors
Small (10MB)
→
Broadcast to all
Executor 1
+ small copy
Executor 2
+ small copy
Executor 3
+ small copy
Executor 4
+ small copy
No shuffle needed!
Sort-Merge Join (large + large)
Both tables shuffled by join key
1. Sort both tables by join key
2. Shuffle matching keys to same partition
3. Merge matching rows
Network-intensive but handles any size
All Join Types in PySpark
# Sample DataFrames
employees = spark.createDataFrame([
    (1, "Alice", 10), (2, "Bob", 20), (3, "Carol", 30)
], ["emp_id", "name", "dept_id"])

departments = spark.createDataFrame([
    (10, "Engineering"), (20, "Marketing"), (40, "Finance")
], ["dept_id", "dept_name"])

# Inner join (default)
employees.join(departments, "dept_id", "inner").show()

# Left join
employees.join(departments, "dept_id", "left").show()

# Right join
employees.join(departments, "dept_id", "right").show()

# Full outer join
employees.join(departments, "dept_id", "full").show()

# Cross join
employees.crossJoin(departments).show()

# Multiple join conditions
df1.join(df2, (df1.id == df2.id) & (df1.year == df2.year), "inner")

# Force broadcast join for performance
from pyspark.sql.functions import broadcast
employees.join(broadcast(departments), "dept_id").show()

# Or set threshold (10MB default)
spark.conf.set("spark.sql.autoBroadcastJoinThreshold", 50 * 1024 * 1024)  # 50MB

Window Functions

Window functions perform calculations across a "window" of related rows without collapsing them into a single row. Essential for rankings, running totals, and lag/lead patterns.

Window Function Patterns
from pyspark.sql.window import Window
from pyspark.sql import functions as F

# Define window - partition by dept, order by salary
w = Window.partitionBy("dept").orderBy(F.col("salary").desc())

# Rank functions
df = df.withColumn("rank",       F.rank().over(w))
df = df.withColumn("dense_rank", F.dense_rank().over(w))
df = df.withColumn("row_number", F.row_number().over(w))  # unique, no ties
df = df.withColumn("percent_rank", F.percent_rank().over(w))

# Aggregate over window
w_all = Window.partitionBy("dept")  # no orderBy = whole partition
df = df.withColumn("dept_avg", F.avg("salary").over(w_all))
df = df.withColumn("dept_total", F.sum("salary").over(w_all))
df = df.withColumn("salary_pct", F.col("salary") / F.col("dept_total") * 100)

# Running total (ordered window)
w_running = Window.partitionBy("dept") \
    .orderBy("hire_date") \
    .rowsBetween(Window.unboundedPreceding, Window.currentRow)
df = df.withColumn("running_total", F.sum("salary").over(w_running))

# Lag and Lead
w_time = Window.partitionBy("product").orderBy("month")
df = df.withColumn("prev_month_sales", F.lag("sales", 1).over(w_time))
df = df.withColumn("next_month_sales", F.lead("sales", 1).over(w_time))
df = df.withColumn("mom_growth",
    (F.col("sales") - F.col("prev_month_sales")) / F.col("prev_month_sales") * 100)

# Top N per group using window
ranked = df.withColumn("rn", F.row_number().over(w))
top3_per_dept = ranked.filter(F.col("rn") <= 3)
rank() vs dense_rank() vs row_number():
- rank(): 1,2,2,4 (skips after tie)
- dense_rank(): 1,2,2,3 (no skip)
- row_number(): 1,2,3,4 (always unique - arbitrary tie-break)

Complex Types - Arrays, Maps, Structs

Real data often has nested structures - JSON events, arrays of tags, maps of attributes. PySpark handles all of these natively.

Working with Arrays
# Create array column
df = spark.createDataFrame([
    (1, ["python", "spark", "sql"]),
    (2, ["java", "scala"]),
    (3, ["python", "r", "julia", "sql"])
], ["id", "skills"])

# Array operations
df.withColumn("skill_count", F.size(F.col("skills")))
df.withColumn("first_skill", F.col("skills")[0])
df.withColumn("has_python", F.array_contains(F.col("skills"), "python"))
df.withColumn("sorted", F.array_sort(F.col("skills")))

# Explode - one row per array element
df.withColumn("skill", F.explode(F.col("skills"))).show()

# Collect back to array
df.groupBy("dept").agg(F.collect_list("name").alias("members"))
df.groupBy("dept").agg(F.collect_set("name").alias("unique_members"))  # dedup

# Map column
df2 = spark.createDataFrame([
    (1, {"level": "senior", "location": "NY"}),
    (2, {"level": "junior", "location": "SF"})
], ["id", "metadata"])

df2.withColumn("level", F.col("metadata")["level"])
df2.withColumn("keys", F.map_keys(F.col("metadata")))

# Struct - named fields
df3 = df.withColumn("address", F.struct(
    F.lit("123 Main St").alias("street"),
    F.lit("New York").alias("city")
))
df3.withColumn("city", F.col("address.city"))
Parsing JSON and Nested Data
# Parse JSON string column
df_json = spark.createDataFrame([
    (1, '{"name":"Alice","score":95}'),
    (2, '{"name":"Bob","score":72}')
], ["id", "json_data"])

schema = StructType([
    StructField("name", StringType()),
    StructField("score", IntegerType())
])
df_json = df_json.withColumn("parsed", F.from_json(F.col("json_data"), schema))
df_json.withColumn("name", F.col("parsed.name")) \
       .withColumn("score", F.col("parsed.score")) \
       .drop("json_data", "parsed") \
       .show()

# Convert struct back to JSON string
df.withColumn("json_str", F.to_json(F.col("struct_col")))
